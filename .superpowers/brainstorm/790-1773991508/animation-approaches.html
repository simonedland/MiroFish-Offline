Live SMS Activity — Animation Approaches

When an SMS is sent between two agents, the graph connection reacts. Pick the style that feels right.

A
Edge Blink SIMPLE
The connection flashes bright on send The edge pulses with a warm glow when an SMS is sent, then fades back. Clean and immediate — attention goes straight to the link.
B
Traveling Dot DIRECTIONAL
A dot travels from sender to receiver Shows direction clearly — you see who's sending to whom. Nodes pulse on send and receive. Extends the existing animation system.
C
Glow + Dot RICH
Edge glows purple + dot travels A distinct purple color (different from existing orange Reddit actions) marks SMS activity with both an edge glow and a traveling particle.
Note: All approaches also need

Wiring the SMS event poll (getSmsEvents) into SimulationRunView — the animation is only the visual part. The plumbing is the same regardless of which style you pick.
